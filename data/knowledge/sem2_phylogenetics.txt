	    
What is Phylogenetics?
Phylogenetics is the science of the evolutionary relationships between (and within) species, although the term  is now used to include a suite of broader issues, such as estimation of evolutionary rates, dating divergence times, reconstruction of ancestral character states and/or quantifying adaptation. While we have defined phylogenetics as applying to species, the same techniques can be applied to families of genes for example.
Within this tutorial, we will be focusing primarily on  molecular phylogenetics, which is interested in how molecules such as DNA or proteins have changed temporally, allowing us to determine evolutionary relationships between groups (of organisms or families of genes). Observing these molecules under certain models of evolution, produces a  phylogeny  or  phylogenetic tree, which is a visual representation depicting a hypothesis about the evolutionary relationships between the taxa included.
Phylogenetic Trees
Phylogenetic trees represent the evolutionary relationships between a group of organisms, genes etc... The branching patterns within the tree represent how these elements evolved from a series of common ancestors.

In Fig. 1 we see a phylogenetic tree with five species (A, B, C, D & E), which are represented at the tips of the branches. Branches are connected at branch points, or nodes. These represent the most recent common ancestor between the two branches. We can think of these nodes as divergence events (from a single group into two groups). For example, in Fig. 2 we can see that the node connecting the branches leading to A and B represents their most recent common ancestor. In the same way, The node directly after the root represents the most recent common ancestor of all species in the tree. We can think about relatedness of any two given individuals in the tree in terms of how recent their most common ancestor is (i.e. their connecting node). We 'read' a phylogenetic tree backwards, and so in Fig. 3 you can see that the MRCA (most recent common ancestor) of species A & B is more recent than that of species B & C. This means A & B are more closely related. however based on the tree shown in Figs. 1-3 we cannot distinguish whether species A & B are more or less closely related than species C & D. The reason for this is that while the vertical axis of the tree represents evolutionary time, the horizontal axis doesn't represent time (at least not in a direct way).
One slightly confusing element of phylogenetic tree interpretation is that the branches can freely rotate (i.e. the order that the species are represented at the branch tips is not fixed). This is illustrated nicely in Fig. 4.

Algorithms used to infer phylogenies
1. Distance-based Methods
Distance-based methods compute an  evolutionary distance, which is the number of changes that have occurred between two species considered to diverge from a common ancestor. Distance based methods (i.e.  Neighbour-joining; NJ) suffer from lack of accuracy when dealing with large volumes of data and distant relationships (i.e. this can make it problematic for sequence data).
2. Maximum Parsimony (MP) methods
MP methods infer a tree that minimizes the total number of  mutations  required to explain the data. Under the maximum parsimony criterion, the shortest possible tree that explains the data is considered as the best tree. This best tree is known as the  most-parsimonious tree. Heuristic search is performed to quickly generate the most-parsimonious tree. Since this methods considers the shortest possible tree as the best tree, actual evolutionary changes that have occurred may be underestimated. Again, this can be problematic for sequence data.
3. Probabilistic methods
Probabilistic methods, such as  Maximum Likelihood (ML) and   Bayesian inference, attempt to find a tree that maximizes the conditional or posterior probability of observing the data. Phylogenetic studies at present, widely utilize Bayesian frameworks due to the possibility of account for the phylogenetic uncertainty, availability of efficient algorithms and their implementation as various computer programs.
Because the aim of this tutorial is to get you familiar with dealing with phylogenetic trees in python, we will focus on NJ and MP trees. While I wouldn't encourage you to do this for your actual science, familiarising yourself with how we deal with this type of data is meaningful.

So we have a tree, is this the end of the story? Actually, there may be multiple possible trees that can be generated from an alignment. We need to understand which of the trees is most representative, and then understand more about how robust the branch relationships are (i.e. are two grouped species always together in a clade? Or only some of the time?).
To do this, we will use a process call bootstrapping, where we generate a large number of trees, and then use all of those trees to construct a consensus tree which will tell us how well supported each of the nodes are.
First we import the relevant modules:
from Bio.Phylo.Consensus import *


Then we generate the replicate alignments using bootstrap:
phylos = bootstrap(phylo, 100)


Note. You can see here that we are running the process we used about to create the first alignment, and repeating it 100 times.
We then produce the trees:
# Select model for distance calculator
calculator = DistanceCalculator('blosum62')
# Prep the calculator
constructor = Phylo.TreeConstruction.DistanceTreeConstructor(calculator)
# Run the bootsrapping process
trees = bootstrap_trees(phylo, 100, constructor)


The variable 'Trees' now contains a list of 100 phylogenetic trees. This lease us with the issue of how we compare each of these trees to understand which node relationships are most frequent, how frequent these relationships are etc...
So, we can take this list of 100 replicate trees, and make a consensus tree. A tree which represents the majority character at every node. We do this in the following way:
consensus_tree = bootstrap_consensus(phylo, 100, constructor, majority_consensus)


Note. Here we give the initial alignment, 100 replicates, the variable 'constructor' which is our distance tree construction calculator, and tell it that we want the 'majority consensus' (read more here
