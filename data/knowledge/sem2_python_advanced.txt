Functions – the basics

Functions are global “store and reuse” code that can be called at any time (or at any point within your code) to perform a predefined action.

There are two types of functions within Python:

Built-in functions that are provided either as part of Python (i.e. print(); float(); str() etc...) or distributed for import into Python (i.e. BioPython)

Functions that we define ourselves and then use


Note. names associated with built-in functions should not be used as variable names.

Alternate ways to import

Importing with a variable name:








As with almost all aspects of Python, you can assign a variable name to an imported module. This can be useful for several reasons, but in most cases it allows you to shorten or simplify long module names.

Writing a function

It is also possible to use a function within a function:










The function kelvin_to_fahr() uses both our original function (celsius_to_fahrenheit()) and kelvin_to_celcius() by simply calling those functions when required.

def kelvin_to_celcius(temp):
    return (temp – 273.15)

def kelvin_to_fahr(temp_k):
    temp_c = kelvin_to_celcius(temp_k)
    temp_f = celcius_to_fahrenheit(temp_c)    
    return temp_f

‘Further Python functionality’

Return vs. Print

It is important to note that within a function return and print are not interchangeable.

return makes a statement visible to the program.

print makes a statement visible to the users.

For example:

def multiply(a, b):
    return (a * b)

In [1]: A = multiply(15, 27)

In [2]: print(A)

Out[2]: 405

def std_dev(sample):
    sample_sum = 0
    for value in sample:
        sample_sum += value
    sample_mean = sample_sum / len(sample)
    sum_squared_devs += (value – sample_mean) * (value – sample_mean)
    return numpy.sqrt(sum_squared_dev / (len(sample) – 1))
