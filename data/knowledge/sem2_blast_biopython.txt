  
Why BLAST?
BLAST is one of the fundamentals of computational biology. At it's heart, it is a database searching tool, that gives us the ability to recognise and quantify sequence homology (with statistical support!) for either nucleotides or amino acides. This is incredibly useful for a wide range of applications, for example BLAST can be used to infer functional and evolutionary relationships between sequences as well as help identify members of gene families.
There are several distinct ways to run Blast depending on your intended use:
1) BLAST online 
 (Links to an external site.)
via the NCBI repository.
This is perhaps the simplest way to run BLAST. That said, it requires a lot of manual steps, and so for large numbers of samples (or integration and reproducibility) it isn't ideal.
2) Standalone BLAST - See here 
 (Links to an external site.)
for more information about installing BLAST locally.
This option allows you to interact with BLAST via your commandline shell. This is useful because it allows for both integration and reproducibility. There are two principle ways that standalone BLAST works. You can either create your own database to BLAST against, or download specific BLAST databases. These can be large though.
3) API BLAST.
This option allows you to interact with BLAST's online databases through either your commandline shell, or, as we will learn in this tutorial, through Biopython.
A note on biopython.
While this will not be your first exposure to biopython as a tool ensuring that you understand the basics is still important. Biopython is perhaps one of the most powerful available to computational biologists. I do like to remind you all "not to reinvent the wheel" buy rewriting scripts which have been written and implemented well by others. most of the time, I'll be referring to functionality within biopython. This tutorial is not going to cover the breadth of things biopython can do, so I'd encourage you all to really read the documentation, here .


Installing Biopython 
Installing biopython.
If you don't already have biopython installed on your machine, there are several ways you can install it. For example via your package manager (i.e. Anaconda), or via direct download from the biopython website 
 (Links to an external site.)
.
Perhaps the easiest way however is to install it via 'pip' (the package installer that ships with Python):
pip install biopython


Note. The above code assumes you have only a single version of python on your system. Depending on your user profile privileges, you may need to use the 'sudo' command (to grant root access).
You can then test whether the install worked correctly by importing the module into a python session:
import Bio


Assuming that you don't get an error here, biopython should be installed correctly.
 


Importing sequences through biopython.
We have spent a lot of time in our workshops writing functions that will parse, amongst other things, FASTA and FASTQ files. We have done it this way, because it is essential that you understand how to do it. In the 'real world', you will probably do it via biopython (mainly because it's really well implemented).
So to import (or parse correctly) a sequence through biopython, you need to do the following steps:
1) Import the specific functions from biopython.
from Bio import SeqIO


2) Use the function ('SeqIO') to parse a file containing sequences.
for record in SeqIO.parse(open(file, "r"), "fasta"):
    print(record.id, record.seq)


Note. The above code is a little different from the way we have done things in the past (i.e. opens the file as part of the 'for loop' statement. It works exactly the same way. Chose the way you feel most comfortable.
The output of this code (the print command), should look something like the example below:
('AT1G04130.1_pop1_+_m', Seq('ATGGCGCTATGGATGGACGCTGGAGCGACGCCTACAACGGAGAACGAAAAGGCT...TAG', SingleLetterAlphabet()))


You will notice that unlike the method by which we have been returning sequences, the 'seq' element contains slightly more data than simply the sequence. If you want to return the actual sequence, you could amend the code to the following:
print(record.id, str(record.seq))


This will print the sequence as a string in the way we familiar with.

5) Save the output to an XML file.
Printing the XML BLAST output to your stdout is perhaps not so useful when we think about reproducibility and robustness. You can modify your code to save the output in the following way.
with open("Blast_output.xml", "w") as f_out:
    result_handle = NCBIWWW.qblast("blastn", "nt", str(record.seq))
    blast_result = result_handle.read()
    f_out.write(blast_result)


Note. Producing output files like this is super useful when dealing with multifasta files containing lots of sequences. Be aware though, accessing BLAST online like this can be time consuming depending on connection speed and server load. It might take a while for this code to generate results.
