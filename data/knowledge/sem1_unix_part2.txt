Uniq

Removing duplicate entries in a file can be achieved using the Uniq utility. The program compares adjacent lines within the input files, and writes a single copy of each entry to the output.

Note. Repeated lines in the input are only detected if they are adjacent. For this reason, Uniq is often paired with sort.

Typical usage:

Usage: uniq [-c | -d | -u] [-i] [-f num] [-s chars] 
[input_file [output_file]]

Pipes

A potentially better solution, uses Pipes. A Pipe in UNIX is a one-way system of passing information from one program to another.

Here, we pipe the sort output from the stdout into Uniq. This produces the same output as using two commands, but doesn’t create additional files.

$ sort new_data_file | uniq

Never 59
Sort 1
Test 10
Test 3
Try three

Cut

The Cut utility is capable of extracting parts of lines from each file to the stdout. The location of the cut section can be defined by:

Bytes (‘-b’)
Characters (‘-c’)
A delimiter (‘-d’)
Field (‘-f’)

usage: cut -b list [-n] [file ...]
       cut -c list [file ...]
       cut -f list [-s] [-d delim] [file ...]

$ man cut

‘Further UNIX commands’

Cut optionality

For each of the cut flag options, a range must be specified.

N	Nth byte, character or field counted from 1
N-	From Nth byte, character or field to end of line
N-M	From Nth to Mth (included) byte, character or field
-M	From first to Mth (included) byte, character of field

For example, to selected the first 5 characters per line one can use:

$ cut –c -5 new_data_file

Combining flags in grep

Option flags can be combined within grep (and other programmes/commands) to allow you to get the exact optionality you want. For example, you may want to know how many lines do not feature a non-case sensitive regex. This involves combining the following:
‘-c’	display the number of matched lines
‘-i’	ignore case sensitivity
‘-v’	display lines which do not match

$ grep –civ test new_data_file

3

Be careful though, some optionality may not be compatible. For example, with the ‘-H’ flag, grep will give the file name:

‘Further UNIX commands’

Using grep on directories

Grep is a very powerful tool. Not only can it search within large text files to find your regex, but it can also search directories and find any file which contain it.

Be careful with such a broad search (i.e. ‘*’ – everything). Depending on the directory you are currently in, it could take a long time to finish! You may also see a lot of the following error message:

$ grep –l Test *

new_data_file

grep: /Users/andrew/Applications: Is a directory

‘Further UNIX commands’

Grep and regular expressions

^ (caret) – match expression at the start of a line


$ - match expression at the end of a line


\ - turn off the characters special meaning (i.e. to search for a $)


[] – match any one of the enclosed characters


[0-9] – use a ‘-’ to indicate a range

$ grep ^Test new_data_file

$ grep Test$ new_data_file

$ grep \$ new_data_file

$ grep [aeiou] new_data_file

$ grep [1-5] new_data_file

‘Further UNIX commands’

Making the script into a program

To make the script into a program (i.e. not have to tell the computer how to run it), you need to do the following:
Look at the current permissions

Change the permissions to make it executable

$ ls –l sequence_count_script.sh

-rw-r--r--  1 andrew  staff  sequence_count_script.sh

$ chmod +x sequence_count_script.sh

$ ls –l sequence_count_script.sh

-rwxr-xr-x  1 andrew  staff  sequence_count_script.sh

Run the program

It should produce exactly the same results as the first time.
